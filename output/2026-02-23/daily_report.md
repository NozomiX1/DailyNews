### 1. [智谱AI] GLM-5: 异步强化学习框架Slime与DSA工程实践
**来源**: 机器之心 | **时间**: 2026-02-23 10:28
**价值**: 🌟🌟🌟🌟 **标签**: [发布] [技术] [研究]
**链接**: https://mp.weixin.qq.com/s/Qmg6iOYo2BXB9qFf4TOpsg

> 🎯 **一句话摘要**：GLM-5 通过异步强化学习框架 Slime 解耦训推流程，并针对 DSA 架构进行工程化修复，显著提升了长程 Agent 任务的训练效率与稳定性。

#### 🔹 核心技术/实现逻辑

- **DSA 架构的工程化适配**：
  - 采用了 **DeepSeek Sparse Attention (DSA)** 以平衡长文本推理与成本，支持 744B 总参数（40B 激活）和 28.5T 训练 token。
  - **Muon Split 机制**：针对 DSA 与 MLA、Muon 优化器叠加时的性能退化问题，将矩阵拆分为不同头的更小矩阵，分别进行矩阵正交化，允许不同尺度更新。
  - **MLA-256 变体**：将 Head Dimension 从 192 提升至 256，减少 1/3 注意力头数，在保持训练计算量不变的同时显著降低解码计算量。
  - **MTP 参数共享**：训练阶段共享 3 层 MTP 参数，提升 token 接受率。

- **异步强化学习基础设施**：
  - **架构解耦**：推理引擎与训练引擎部署于不同 GPU，异步并行运行。推理持续生成轨迹，训练持续消费更新，解决长轨迹导致的 GPU 空转问题。
  - **TITO Gateway**：直接截获推理引擎产生的 Token ID 和元数据，绕过文本中转，确保训练端采样与推理端完全一致，避免 RL 信号错位。
  - **双侧重要性采样**：复用 rollout 时的 log 概率作为行为策略代理，并采用**双侧 Token 级掩码**（超出比率区间的 Token 梯度置零），解决异步训练中策略追踪难题。

- **训练稳定性与 Agent 算法**：
  - **确定性算子替换**：将 DSA 中非确定性的 CUDA top-k 算子替换为 `torch.topk`，虽然牺牲微小速度，但恢复了 RL 训练的稳定性并提升性能；同时冻结索引器参数。
  - **三级奖励体系**：针对 HTML 幻灯片生成，设计了静态规则（布局/字体）、运行布局（DOM 节点指标）、视觉感知（视觉层检测）三级奖励。
  - **跨阶段蒸馏**：在推理 RL、Agent RL、通用对齐 RL 三阶段间引入对数概率对比，防止灾难性遗忘。
  - **分层上下文管理 (HCM)**：针对超长上下文，优先折叠早期记录并保留最近 5 轮记忆，若超限则清空工具历史重置。

#### 📊 实验数据/关键结论

- **Agent 任务基准**：在 SWE-bench、Terminal-Bench、BrowseComp、MCP-Atlas 等基准上达到开源 **SOTA**。
- **上下文管理效果**：采用 HCM 策略后，BrowseComp 任务准确率从 **55.3% 提升至 75.9%**。
- **审美奖励效果**：三级奖励体系将 16:9 幻灯片合规率从 **40% 提升至 92%**。
- **综合对比**：在 Vending-Bench 2 和 CC-Bench-V2 上表现接近或超越 Claude Opus 4.5。

#### 💡 独家洞察/局限性

- **工程价值**：GLM-5 的技术报告展示了大量“脏活累活”式的工程 Trick，如解决 CUDA 算子不确定性、Token 对齐问题等，这些往往是论文不写但工业界落地必须解决的痛点。
- **范式转移**：从“同步 RL”转向“异步 RL”是处理长程复杂任务的必经之路，Slime 框架为此提供了标准化的解决方案。
- **局限性**：虽然 DSA 降低了成本，但在引入 MLA 和 Muon 优化器时遇到了复杂的兼容性问题，说明新型稀疏架构的工程门槛极高。

### 2. [上海交大] SIE框架：结构化上下文环境激活LLM通用推理泛化
**来源**: 机器之心 | **时间**: 2026-02-23 10:28
**价值**: 🌟🌟🌟🌟 **标签**: [ICLR 2026] [强化学习] [大模型推理] [研究]
**链接**: https://mp.weixin.qq.com/s/5esxbT_fd8waZRfKnbaOFg

> 🎯 **一句话摘要**：上海交大提出 SIE 框架，利用知识图谱等结构化数据低成本构建可验证的 RL 训练环境，有效解决了 RL 环境扩展瓶颈，并在无人工标注的情况下显著提升了模型的通用推理与泛化能力。

#### 🔹 核心技术/实现逻辑

- **结构化上下文环境 (SIE)**：将静态的结构化数据（如 Freebase 知识图谱）转化为动态的上下文环境。环境通过“软约束”形式嵌入 Prompt，模型的推理过程被建模为在该上下文中的隐式动作序列。

- **自动化环境构建流水线**：
  - **支撑子图提取**：针对 QA 对，从问题实体和答案实体出发进行双向多跳检索，利用最短路径算法提取连接路径，确保推理链条的完整性。
  - **干扰子图过滤**：为防止模型通过关键词匹配“作弊”，引入基于 **Cross-Encoder** 的语义重排序策略，筛选出与问题语义高度相关但实际为错误路径的高难度干扰项，迫使模型进行深度逻辑推理。
  - **部分可观测性**：构建信息密度递减的“Partial SIEs”（如保留 100%、75%、50%... 的支撑信息），人为制造信息缺失困境，迫使模型利用内部知识进行推理。

- **RL 训练策略**：使用 **GRPO (Group Relative Policy Optimization)** 算法。引入双重奖励机制：**答案奖励**（验证最终输出正确性）与 **格式奖励**（约束输出格式）。

#### 📊 实验数据/关键结论

- **领域内推理提升**：在 WebQSP 数据集上，相较于无结构化上下文的 RL 基线，模型性能平均提升 **30% 以上**。

- **RL 显著优于 SFT**：
  - **RL w/ SIE** 平均增益：**53.7%**
  - **SFT (Supervised Fine-tuning)** 平均增益：**11.4%**
  - 结论：通过试错学到的策略比单纯模仿轨迹更深刻。

- **通用推理零样本泛化**：
  - **Qwen2.5-7B** 在 **GSM8K**（数学推理）上：29.2% → **87.4%**
  - 在 **MATH500** 和逻辑谜题上同样表现出显著提升，证明了结构化推理技能可迁移至通用领域。

- **鲁棒性验证**：即便在信息极度缺失（SIE-0%，仅保留干扰项）的情况下，RL 微调依然能带来性能提升，模型学会了利用干扰项作为负向约束。

#### 💡 独家洞察/局限性

- **核心洞察**：该工作打破了“结构化推理难以泛化”的传统认知。通过在结构化环境中学习多跳组合推理的底层认知模式，模型能够将这种能力迁移到数学和逻辑等非结构化任务中。
- **工程价值**：提供了一条不依赖昂贵专家标注（如数学题人工解答）的 RL 路径，利用全球海量的结构化数据（KG、表格、数据库）即可自动化构建高质量 RL 训练场，极大地降低了后训练成本。
- **局限性**：目前的实验主要基于知识图谱数据，对于其他类型结构化数据（如代码 AST、商业数据库）的效果有待进一步验证。

### 3. [MiniMax] M2.5: 原生Agent强化学习系统Forge与3T调用量验证
**来源**: 机器之心 | **时间**: 2026-02-23 12:01
**价值**: 🌟🌟🌟🌟 **标签**: [发布] [Agent架构] [RLHF] [Benchmark]
**链接**: https://mp.weixin.qq.com/s/u1Qc6TO02qQLwdtujh9VpA

> 🎯 **一句话摘要**：MiniMax M2.5 通过原生 Agent 强化学习系统 Forge 实现了训练与推理的工程化突破，以 SOTA 编程能力与极低成本激活了大规模 Agent 应用需求。

#### 🔹 核心技术/实现逻辑

- **Forge 架构解耦**：引入中间件抽象层与 Gateway Server，将 Agent 的高层行为与底层模型复杂性隔离；利用 Data Pool 异步收集训练轨迹，使生成与训练彻底解耦，无需修改 Agent 内部代码即可接入多种框架。
- **Prefix Tree Merging**：将线性训练样本重构为树形结构，消除多轮 Agent 请求间大量重复的上下文前缀，实现了约 **40 倍** 的训练加速，并显著降低显存开销。
- **Windowed FIFO 调度**：在最大化系统吞吐量的同时，通过滑动窗口约束控制样本的离策略程度，防止训练分布向“快而简单”的样本严重偏移，兼顾效率与稳定性。
- **复合奖励机制 (RL)**：针对长轨迹信用分配难题，设计了由 **过程奖励**（中间行为密集监督）、**任务完成时间奖励**（激励并行策略与最短路径）与 **Reward-to-Go**（降低梯度方差）组成的复合奖励。
- **上下文管理整合**：将上下文管理机制直接整合进 RL 交互循环，视为驱动状态转换的功能性动作，从训练阶段解决长程任务中的注意力稀释问题。

#### 📊 实验数据/关键结论

- **SWE-Bench Verified**: 80.2%（通过率，达到与 Claude Opus 相当的水平，Multi-SWE-Bench 排名第一）。
- **独立测评**: Simon Willison 使用 mini-swe-agent 测评排名第三（开源模型中第一，仅次于 Claude 4.5 和 Gemini 3 Flash）。
- **推理吞吐**: SemiAnalysis 实测在 8 张 H200 显卡上，单卡吞吐量约 **2500 tokens/s**。
- **市场表现**: OpenRouter 周调用量暴涨至 **3.07T tokens**，带动 100K-1M 长文本区间需求。
- **成本**: 100 TPS 版本输入 $0.3/M，输出 $2.4/M；50 TPS 版本输出价格减半。

#### 💡 独家洞察/局限性

- **工程决定落地**：M2.5 的成功不仅在于模型能力，更在于 Forge 系统解决了 Agent 训练中的“数据重复”和“调度漂移”等工程痛点，这为后续 Agent 模型的训练提供了重要的架构参考。
- **长文本的刚需释放**：M2.5 的火爆验证了一个市场假设：阻碍 Agent 落地的不仅是智力，还有长上下文场景下的成本与速度。当性能与成本达到临界点，被压制的工具调用需求会呈指数级释放。

### 4. [华为] CLI-Gym: 环境反演数据生成管线与Terminal-Bench优化
**来源**: 机器之心 | **时间**: 2026-02-23 12:01
**价值**: 🌟🌟🌟🌟 **标签**: [发布] [Agent] [数据工程] [微调]
**链接**: https://mp.weixin.qq.com/s/ty1gzYIV_CpRkKLBbJSsMg

> 🎯 **一句话摘要**：华为提出 CLI-Gym，通过“环境反演”技术自动构建 CLI 任务数据，微调后的 LiberCoder 模型在 Terminal-Bench 上实现最高 28.6% 的解决率提升。

#### 🔹 核心技术/实现逻辑
- **环境反演**：不同于传统人工构造故障环境，CLI-Gym 从健康的 Docker 镜像（包含通过所有 Unit Tests 的环境）出发，利用“破坏者” Agent 执行“劣化”操作（如删除依赖、修改权限、篡改配置），主动制造环境故障。
- **自动任务构建**：系统监测到 Unit Tests 失败后，结合失败日志、执行轨迹和环境变更，自动生成问题描述和修复目标，将其转化为可修复的 CLI 任务实例。
- **核心抽象与可复现性**：以 Codebase、Dockerfile 与 Base Image 为核心抽象，定义 CLI Coding 实体。这种基于 Docker 的结构化定义确保了环境的可复现性与版本控制。
- **数据规模化**：在 29 个真实开源仓库基础镜像上，自动化生成了 1,655 个环境密集型 CLI 任务，远超原有人工构造数据的规模，且无需人工标注。

#### 📊 实验数据/关键结论
- **Terminal-Bench 1.0 Pass@1**：
  - LiberCoder-32B：+28.6% (至 38.9%)
  - LiberCoder-235B-A22B：+21.1% (至 46.1%)
- **数据质量**：收集了 291 条高质量成功修复轨迹，每个任务平均包含超过 20 个失败测试，提供了丰富的诊断信号。
- **泛化能力**：模型在软件工程、系统管理、安全修复与调试等多个任务类别上均有提升，说明数据有效增强了模型的环境交互泛化能力。

#### 💡 独家洞察/局限性
- **解决行业痛点**：针对当前 Agentic Coding 研究过于集中在代码生成（SWE-bench），而忽视了环境运维（依赖、配置、权限）类问题的瓶颈，填补了该领域公开数据 Scaling 方案的空白。
- **工程范式创新**：将数据生产管线本身建模为一种 Agentic Coding 任务，不仅适用于 CLI 任务，其框架也兼容此前的 SWE 系列方法，具有很好的通用性。
- **局限性**：虽然实现了自动化，但任务生成依赖于 Unit Tests 的覆盖率和质量；如果“破坏者” Agent 能力不足，生成的故障类型可能受限。

### 5. [王捷] AI经济测度：GDP/Token 生产函数与经济图灵测试
**来源**: 机器之心 | **时间**: 2026-02-23 18:51
**价值**: 🌟🌟🌟🌟 **标签**: [研究] [架构] [经济学]
**链接**: https://mp.weixin.qq.com/s/QWpPtAfmr-gj96AHSVWuIQ

> 🎯 **一句话摘要**：提出 AI 生产能力函数 $C(M)$，以 GDP/Token 为核心指标，结合“经济图灵测试”填补了从模型技术能力到宏观经济产出之间的度量空白。

#### 🔹 核心技术/实现逻辑

文章提出了一种新的评估框架，将 AI 模型视为一种新型生产要素，旨在解决传统 Benchmark（如 MMLU）无法衡量经济价值的问题。

*   **AI 生产能力函数 $C(M)$ 定义**：
    $$C(M) = \frac{\sum V_j \times P_j}{\sum T_j}$$
    *   **分子（经济产出）**：任务 $j$ 的经济价值 $V_j$ 与其被成功完成且通过验收的概率 $P_j$ 的乘积之和。
    *   **分母（Token 投入）**：完成任务所消耗的总 Token 数。值得注意的是，分母中的单任务 Token 消耗 = 平均单次 Token 消耗 / 成功率，意味着成功率越高，实际单位任务成本越低。

*   **经济图灵测试**：
    引入了 **ETT (Economic Turing Test)** 概念。传统的技术正确性并不等于经济有效性。只有当 AI 的输出在真实经济环境中被人类或组织接受时，该产出才计入 GDP。ETT 相当于一个“制度与偏好约束”，过滤掉技术上正确但无用的产出。

*   **任务价值异质性**：
    打破传统 Benchmark 中所有任务权重相同的假设。$C(M)$ 自动对高价值任务（如核心医疗决策、金融交易）赋予更高权重，对低价值任务（如简单闲聊）降权。

#### 📊 实验数据/关键结论

本文为理论框架构建，未提供具体的模型跑分数据，但得出了以下关键结构化结论：

*   **度量转化**：实现了从“点状性能”到“任务空间积分”的转变，即从单一任务的正确率转向整个经济任务空间中的价值加权期望产出。
*   **AI 的经济属性**：AI 被定义为“可资本化复制的任务执行能力”，兼有劳动（任务完成）和资本（规模复制）的属性。
*   **产业渗透预测**：
    *   **高潜力行业**：软件开发、数字营销、在线客服（高数字化、易形式化、ETT 约束宽松）。 
    *   **低潜力行业**：高风险医疗、复杂现场作业（ETT 约束严格、环境非数字化）。

#### 💡 独家洞察/局限性

*   **技术点评**：该框架为“Tokenomics”提供了坚实的理论基础。它不仅仅关注模型“能不能做”，更关注“做得值不值”和“人类买不买账”。这对企业选型（不仅是看 Benchmark 排名，而是看 ROI）具有重要指导意义。
*   **局限性**：目前框架尚处于理论阶段，落地依赖于庞大的数据统计工作，包括构建全社会的“任务集合”以及确定各类任务的“影子价格”。
*   **未来工作**：需要建立标准化的 ETT 判定机制，并积累真实部署数据来校准函数参数。

### 6. [HKUST-GZ] NEF-NET v2: 几何视角变换心电全景与三阶段自适应校准
**来源**: 新智元 | **时间**: 2026-02-23 21:11

> 🎯 **一句话摘要**：通过几何视角变换替代显式场建模，结合Any-pairs预训练与On-the-fly自适应校准，在真实临床场景下实现了高精度的任意导联心电图生成。

#### 🔹 核心技术/实现逻辑

- **架构升级：View-to-View Transformation**
  - 摒弃了 v1 版本中容易导致波形细节丢失的“特征平均融合”策略。
  - **View Encoder**：使用 **1D-ResNet** 提取单导联特征，并引入类似 **FiLM (Feature-wise Linear Modulation)** 的仿射调制机制，根据目标导联角度增强相关特征、抑制无关特征。
  - **Geometric View Transformer (GeoVT)**：核心模块包含 **Geometric Angular Attention (MGAA)**，通过 Cross-Attention 比较目标角度与输入角度的几何相似度权重；堆叠 **View Transformation Blocks** 实现粗到细的特征投影。

- **三阶段训练与部署流程**
  - **Stage 1: Any-pairs Pretraining**：在多数据集上进行“任意记录-任意目标”的动态采样训练，学习稳健的跨视角映射先验。
  - **Stage 2: Device Calibration**：针对特定采集设备进行微调，对齐不同设备的信号域差。
  - **Stage 3: On-the-fly Calibration**：在推理阶段引入可学习的角度偏差参数 `(dθ, dφ)`。利用每条 ECG 的**前 5 秒**信号进行快速自校准，适应电极贴片位置误差和个体解剖差异，剩余时间用于实际生成。

- **Panobench 数据集**
  - 构建了高密度心电数据集，利用 CT 标注每个视角的精确空间角度，用于系统验证全景合成能力。

#### 📊 实验数据/关键结论

- **重建精度提升**：在 CPSC2018 等数据集上，NEF-NET v2 相比 baseline 方法平均 **PSNR 提升 6.9dB**；在房颤 (AF) 信号上 PSNR 提升达 **7.3dB**。
- **临床细节保留**：在 V5 导联回归任务中，v2 模型能准确恢复具有诊断意义的关键波形拐点（如 ST 段偏移），而 v1 或传统方法容易抹平这些细节。
- **泛化能力**：在涵盖 9 类心律/心肌异常的测试中，模型在病理条件下的重建保真度显著提高，正常与异常节律间的性能差距缩小。
- **消融实验**：三阶段流程中，On-the-fly Calibration 显著提升了真实场景（存在电极偏移）下的合成质量。

#### 💡 独家洞察/局限性

- **工程价值**：提出的“前5秒自校准”策略非常巧妙，解决了医疗信号采集过程中普遍存在的操作误差问题，具有很高的落地参考价值。
- **几何约束**：通过显式引入角度 Embedding 和 Attention 机制，模型不仅拟合了数据分布，还学到了心电传播的几何先验，这对少样本场景下的合成至关重要。

### 7. [阿里巴巴] COMI：基于边际信息增益的长文本压缩与粗到细策略

> 🎯 **一句话摘要**：针对现有长文本压缩方法在高压缩率下因保留冗余相似内容导致性能崩塌的问题，COMI 引入“边际信息增益（MIG）”指标与粗到细自适应压缩策略，实现 32 倍压缩下性能反超原始长上下文。

#### 🔹 核心技术/实现逻辑

文章指出现有方法（如只关注 Retriever 的相关性）容易导致“信息内卷”，即保留了大量语义雷同的 Token，干扰模型推理。COMI 提出了以下改进：

*   **边际信息增益**：
    将压缩决策从单维度相关性升级为“相关性-冗余性”双维度权衡。
    *   **公式逻辑**：`MIG = 本单元与查询的相关性 - 与其他单元的最大相似度`
    *   该指标奖励与问题高度相关的片段，同时惩罚与已选内容高度重复的片段。

*   **粗到细自适应压缩**：
    *   **阶段一：粗粒度组重分配**
        将长文本划分为等长片段，基于组间 MIG 动态调整各段的压缩预算。信息密度高、冗余度低的片段（如关键证据段落）获得更宽松的压缩率（保留更多 Token），而重复区域则被大幅压缩。
    *   **阶段二：细粒度 Token 融合**
        在片段内部，根据 Token 级别的 MIG 进行加权融合。高 MIG Token（相关且独特）在融合中占据主导权重，避免传统平均池化导致关键细节被“平滑”掉。

#### 📊 实验数据/关键结论

*   **NaturalQuestions (EM)**：在 32 倍压缩率下，COMI（基座 Qwen2-7B）达到 **49.15** 分，比次优基线高出近 **25 个点**。
*   **对比原生长上下文**：在 Qwen3-4B 上，32 倍压缩后的 COMI 在 NaturalQuestions 上达到 **F1 28.89**，远超直接输入完整上下文的 **16.90**。证明高质量压缩能消除噪声干扰，实现“信息提纯”。
*   **超长文本鲁棒性**：在 32K 超长文本（NarrativeQA）上，仍能稳定保留推理链关键节点。
*   **推理效率**：在 32 倍压缩下，端到端推理速度提升 **2 倍以上**。压缩阶段开销较小（NarrativeQA 任务中压缩耗时 2.76 秒）。

#### 💡 独家洞察/局限性

*   **“减负即增效”的实证**：实验结果有力地反驳了“压缩必然丢失信息导致性能下降”的直觉，证明了在高质量过滤（去除高度相似的冗余噪声）后，模型对核心证据的聚焦能力反而变强。
*   **工程启示**：对于 RAG 系统或长上下文应用，简单地堆砌检索到的片段并非最优解，对检索结果进行“去重”和“多样性筛选”至关重要。

- **测试维度的转变**：从传统的「图灵测试」（欺骗人类）或「标准化考试」（刷题）转向「科学重发现测试」，标志着业界对 AGI 的要求从「模仿」转向「创造」。
- **超级智能的门槛**：马斯克的反驳揭示了一个关键逻辑——如果一个系统能做到人类顶尖科学家无法做到的事，且可无限复制，它本身就超越了「人类水平」的定义，属于 ASI（Artificial Superintelligence）。
- **局限性**：本文主要为观点汇总，未涉及具体的算法改进或实现「科学发现」的技术路径（如 System 2 思维链的具体实现细节）。
