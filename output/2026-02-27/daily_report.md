### 1. [DeepSeek & 清华/北大] DualPath: 智能体推理双路径加载架构与存储带宽优化
**来源**: 机器之心 | **时间**: 2026-02-27 09:21
**价值**: 🌟🌟🌟🌟 **标签**: [发布] [系统架构] [推理优化] [KV-Cache]
**链接**: https://mp.weixin.qq.com/s/vurzKF6HCLryqV2scJAD1g

> 🎯 **一句话摘要**：DualPath 通过引入存储到解码的双路径加载机制，聚合集群存储带宽以解决 PD 分离架构下的 I/O 瓶颈，在智能体高 KV 命中率场景下显著提升推理吞吐量。

#### 🔹 核心技术/实现逻辑

- **背景与痛点**：智能体工作负载呈现“长上下文、短追加”特征，KV-Cache 命中率通常高于 95%。在传统的 Prefill-Decode（PD）分离架构中，KV-Cache 的读取压力全部集中在 Prefill Engine（PE）的存储网卡上，而 Decode Engine（DE）的存储带宽闲置，导致 I/O 不平衡成为性能瓶颈。
- **双路径加载机制**：
  - **Path 1（传统）**：Storage -> PE Buffer -> PE HBM。
  - **Path 2（创新）**：Storage -> DE Buffer -> (RDMA) -> PE HBM。
  - **带宽池化**：系统动态分配两条路径负载，将集群内所有引擎的存储网卡聚合成全局资源池，打破单节点 I/O 限制。
- **以计算网卡（CNIC）为中心的流量管理**：
  - 统一管理 GPU 相关流量（本地内存拷贝与网络传输）。
  - 利用 QoS 机制，将推理通信设为高优先级，确保 KV-Cache 加载流量仅利用闲置带宽，不干扰延迟敏感的推理任务（如 AllToAll, ReduceScatter）。
- **自适应请求调度**：调度器实时监控各引擎的磁盘读取队列长度和计算负载，动态决定每个请求的最优加载路径，并通过计算配额机制优化引擎内调度，减少 GPU 执行气泡。
- **数据块布局**：区分完整块（所有层信息，用于存储交互）与层级块（单层信息，用于流式传输），配合 Layerwise Prefill 解决 HBM 容量瓶颈。

#### 📊 实验数据/关键结论

- **实验环境**：1152 GPU 生产集群，自研推理框架（核心改动约 5000 行代码），底层集成 FlashMLA、DeepGEMM、DeepEP，存储后端为 3FS。
- **评测模型**：DeepSeek 660B (MoE + 稀疏注意力)、DeepSeek 27B、Qwen 32B。
- **离线批量推理 (RL Rollout 场景)**：
  - **DS 660B**: 相比 Basic 基线最高提升 **1.87 倍** 吞吐量（JCT 加速）。
  - **DS 27B**: 最高提升 **1.78 倍**。
- **在线服务**：
  - **DS 660B**: 服务吞吐量（到达率上限）提升 **2.25 倍**。
  - **DS 27B**: 服务吞吐量提升 **1.67 倍**。
  - 在满足 SLO (TTFT ≤ 4s, TPOT ≤ 50ms) 的前提下，DualPath 显著提升了系统承载能力，且未引入额外的解码延迟。

#### 💡 独家洞察/局限性

- **适用场景特定性**：该优化主要针对“智能体”场景（高 KV-Cache 命中率，长上下文，短 Append）。对于传统的长文本生成任务，其优势可能不如智能体场景明显。
- **工程复杂度**：引入了双路径调度、CNIC 流量控制以及复杂的层级数据流管理，大幅增加了推理系统的实现与调试复杂度（论文提到核心改动 5000 行代码）。
- **资源利用平衡**：虽然解决了存储瓶颈，但 DRAM 和 CNIC 面临额外的带宽压力（增加了 DE Buffer 的拷贝开销），文中通过理论分析证明了在 PCIe 拓扑良好的情况下不会形成新瓶颈。

### 2. [上海交大/UC Berkeley] LLM Agent Scaling：信息冗余导致边际效应递减，多样性优于数量
**来源**: 机器之心 | **时间**: 2026-02-27 13:01
**价值**: 🌟🌟🌟🌟 **标签**: [研究] [LLM智能体] [多智能体系统] [架构设计]
**链接**: https://mp.weixin.qq.com/s/aFg6PpQcPXMjd22hLWsG8g

> 🎯 **一句话摘要**：揭示了LLM多智能体系统的扩展瓶颈在于信息冗余，证明增加Agent数量的收益迅速饱和，而通过引入Persona或模型多样性构建的异质系统，能以更少计算成本获得更高性能。

#### 🔹 核心技术/实现逻辑

- **同质扩展失效验证**
  - **实验设置**：使用同一底座模型与系统提示，通过 Vote（单轮多数投票）和 Debate（多轮交互）两种机制协作。
  - **现象**：在 GSM8K、ARC 等 7 个基准任务上，当 Agent 数量 N 从 1 增至 4 时性能提升明显，但继续增加 N 后准确率迅速进入平台期，边际收益接近 0。原因在于同质 Agent 输出高度相关，新增调用仅带来重复信息。

- **多样性配置方案**
  - **L1（Baseline）**：Agent 完全一致。
  - **L2（Persona多样性）**：仅改变角色设定。
  - **L3（Base Model多样性）**：使用不同 backbone 模型。
  - **L4（完全多样性）**：结合不同 Persona 与不同 Model。
  - **结论**：每引入一层多样性，系统性能显著上移，L4 效果最佳。

- **信息论分析框架**
  - **核心定义**：定义了 **“有效信息通道数” (K)**，即系统中不冗余、彼此互补的有效信息源数量。
  - **理论推导**：性能主要由有效信息通道 K 决定，而非单纯的 Agent 数量 N 或推理次数 n。
  - **互补率**：引入新通道对未获取任务证据的覆盖比例。系统性能受 K 与互补率支配，解释了为何同质扩展会遭遇边际效益递减。

#### 📊 实验数据/关键结论

- **效率对比**：在多个任务上，仅使用 **2 个完全异质的 Agent**，即可达到甚至超过 **16 个同质 Agent** 的平均性能。
- **Benchmark 涵盖**：GSM8K（数学推理）、ARC（科学推理）、TruthfulQA（事实性）、HellaSwag（常识推理）、WinoGrande、Pro Medicine 等。
- **理论拟合**：经验成功率与基于有效信息通道的理论预测曲线高度吻合，验证了“信息冗余是瓶颈”的假设。

#### 💡 独家洞察/局限性

- **工程启示**：在进行 Agent 系统设计时，不应盲目堆叠同质模型的调用次数（Scale-out），而应优先追求“推理多样性”。可以通过设计互补的 Persona、混合不同架构的模型（如 GPT-4 + Claude + Llama）或赋予不同工具权限来打破信息冗余。
- **局限性**：目前的理论框架主要基于经验拟合与若干假设，对于如何精确量化特定任务的“有效信息通道 K” 仍需进一步研究。

- 论文地址：https://arxiv.org/pdf/2602.03794
- GitHub 代码：https://github.com/SafeRL-Lab/Agent-Scaling

### 3. [昆仑天工] SkyReels-V4: 双流MMDiT架构与全模态音视频同步生成
**来源**: 机器之心 | **时间**: 2026-02-27 14:34
**价值**: 🌟🌟🌟🌟 **标签**: [多模态] [视频生成] [MMDiT] [音视频同步] [技术报告]
**链接**: https://mp.weixin.qq.com/s/bEi9bIHN0z8ebB89Pei5eQ

> 🎯 **一句话摘要**：首个支持多模态输入及统一生成编辑任务的视频基础模型，通过双流MMDiT与RoPE频率缩放实现原生音画同步，并在Artificial Analysis榜单位居全球第二。

#### 🔹 核心技术/实现逻辑

- **双流 MMDiT (Multimodal Diffusion Transformer) 架构**：
  - 采用对称设计，视频和音频分支拥有独立通道，但共享同一个基于 MMLM 的文本处理“大脑”。
  - **混合注意力机制**：在前 M 层中，保留各自独立的参数空间（如 LayerNorm、MLP），通过联合自注意力进行交互；在随后的 N 层切换到单流架构，使用共享参数提升计算效率。
  - **双向跨注意力**：视频流关注音频特征，音频流反向关注视频特征，在特征层面实现深度咬合。

- **时间尺度对齐**：
  - 引入 **RoPE 旋转位置编码频率缩放**。针对视频（21帧）与音频（218个token，44.1kHz）的时间流速差异，将音频 RoPE 频率按比值约 0.09633 进行调整，使其匹配视频的时间分辨率。
  - 采用带偏移的 **3D RoPE**，为条件潜变量赋予负数时间索引，区分输入条件与生成目标。

- **通道拼接统一框架**：
  - 将噪声视频潜变量、VAE 编码的条件帧及二进制掩码在通道维度直接拼接。
  - 将不同任务（T2V, I2V, 视频扩展, 局部重绘, 主体替换）简化为特定掩码配置下的修复问题。

- **高效生成策略**：
  - **VSA (Video Sparse Attention)**：两阶段分层处理。先粗略聚合时空块定位关键区域，再对 Top-K 关键块进行密集注意力计算。在保证画质前提下，将注意力计算成本降低约 3 倍。
  - **超分与插值**：联合生成低分辨率全序列与高分辨率关键帧，再由专用模块补充细节。

- **多阶段渐进式训练**：
  - 从 256px T2I 预训练（30亿张图像）起步，逐步引入视频片段，分辨率阶梯式上升至 480p、720p、1080p。
  - 最终使用 500 万条多模态视频 + 100 万条精选高质量视频进行 SFT（有监督微调）。

#### 📊 实验数据/关键结论

- **Artificial Analysis 榜单**：
  - 文本生成视频（T2V，含音频）现役模型榜单：**全球第 2**。
  - 全球历史所有 T2V 模型总榜：**第 4**。
  - 超越模型：Veo 3.1, Sora 2, Vidu Q3, Wan 2.6。
- **生成规格**：支持 **1080p** 分辨率、**32 FPS** 帧率、最长 **15 秒** 输出。
- **推理效率**：通过 VSA 稀疏注意力机制，注意力计算成本降低 **约 3 倍**。

#### 💡 独家洞察/局限性

- **全栈式整合价值**：SkyReels-V4 最大的工程价值在于打破了传统视频生成流程中“生成、编辑、配音”分离的孤岛，将多模态参考（文本、图、视频、音频）整合进单一网络，实现了“瑞士军刀”式的端到端创作。
- **局限性**：根据文中 Demo 描述，模型在**生成画面中的文字**（OCR 能力）方面仍有待提升。
- **技术意义**：RoPE 频率缩放的具体数值（0.09633）为解决多模态扩散模型中的异构时间对齐问题提供了重要的工程参考。

### 4. [UIUC DREAM Lab] SIPDO: 基于合成数据反馈的闭环自进化 Prompt 优化
**来源**: 机器之心 | **时间**: 2026-02-27 22:12
**价值**: 🌟🌟🌟🌟 **标签**: [研究] [Prompt Optimization] [LLM] [Agent]
**链接**: https://mp.weixin.qq.com/s/6OFYbFZKfURXyda4aFEtfQ

> 🎯 **一句话摘要**：SIPDO 提出了一套闭环自进化框架，通过主动生成高难度合成数据来“攻击”当前 Prompt，并结合局部与全局回归验证机制，实现了 LLM Prompt 在动态环境下的持续自我修正与性能提升。

#### 🔹 核心技术/实现逻辑

文章将 Prompt Optimization 的发展脉络与参数学习的历史进行了对标（遗传算法 -> SGD -> Adam），重点解析了 **SIPDO (Self-Improving Prompt via Data Optimization)** 这一“Beyond First-Order”阶段的代表作。

*   **SIPDO 双 Agent 协作架构**
    系统由两个核心 Agent 组成，形成对抗与修复的闭环：
    1.  **Data Generator (进攻方)**：负责生成能暴露 Prompt 弱点的合成样本。
        *   **先定 Label 后生成 Input**：从先验分布 $p^*(y)$ 抽取目标 Answer，再生成 Question，避免语义错配。
        *   **Three-voter check**：引入 3 个专家 Agent 核验生成数据的 Question-Answer 一致性与基本事实正确性。
        *   **Latent Template**：从 few-shot 示例中抽取题型骨架（Latent Variable），保证结构对齐真实分布。
        *   **Curriculum Generation (难度递进)**：利用 `summarizer` 将上一难度的输出摘要作为下一层的 latent cue，实现语义深度的累积。
    2.  **Auto Prompt Optimizer (防守方)**：基于失败样本进行 Prompt 修复。
        *   **Error Analysis**：显式识别失败模式，形成 `error slice`。
        *   **Recommendation**：利用 reflection module 生成 `textual patch`（具体的修改指令）。
        *   **Refinement**：将 patch 写入 Prompt 并重排结构。

*   **双重回归验证机制 (Regression Control)**
    为了防止“修了新错，犯了老错”，SIPDO 设计了严格的确认流程：
    *   **Local Confirmation**：仅在当前新产生的 error slice 上测试修复后的 Prompt。
    *   **Global Confirmation**：将修复后的 Prompt 放入**所有历史已解决样本**中全量回测。只有当历史回退率为 0 时，才接受本次修改。这一机制有效地抑制了 Prompt 优化中的灾难性遗忘。

#### 📊 实验数据/关键结论

*   **Benchmark**：在 BIG-Bench 的 6 个任务上进行测试。
*   **对比基线**：CoT (Chain-of-Thought), APE, PromptAgent。
*   **结果表现**：SIPDO 在多数任务与不同模型上 **consistently outperforms** 标准基线，证明了通过合成数据反馈进行闭环优化的有效性。

#### 💡 独家洞察/局限性

*   **方法论对标**：文章指出 Prompt Optimization 正在复刻 Parameter Learning 的历史。目前的创新点在于将成熟的参数优化思想（如 Curriculum Learning、Momentum、二阶信息）迁移到离散的文本空间。
*   **工程启示**：SIPDO 提供的可复用 Prompt Templates（含 Error Analysis、Recommendation 等）对构建自动化 Agent 调优系统具有直接的参考价值。
*   **局限性**：合成数据的质量高度依赖于 Generator 的能力，且生成过程需要调用多次 LLM，计算成本较高。

*   博客: [Evolution of Prompt Optimization (DREAM Lab)](https://dream.ischool.illinois.edu/blogs/evolution_of_prompt_optimization.html)

### 5. [UCSB/NUS/Berkeley] DevOps-Gym: 端到端全链路基准与Agent 0%成功率
**来源**: 新智元 | **时间**: 2026-02-27 14:15
**价值**: 🌟🌟🌟🌟 **标签**: [研究] [Benchmark] [AI Agent] [DevOps]
**链接**: https://mp.weixin.qq.com/s/X5w6CDMkkIwomQ4a93rpFg

> 🎯 **一句话摘要**：首个针对真实DevOps全生命周期的基准测试，暴露SOTA模型在构建、监控及长链路任务中存在0%成功率的致命短板，揭示了当前AI Agent缺乏动态系统理解与长程推理能力。

#### 🔹 核心技术/实现逻辑

- **四大核心阶段**：覆盖 DevOps 完整生命周期的四个关键环节：
  - **Build & Configuration**：处理依赖冲突、构建工具（如Maven/Gradle）迁移及工具链升级，要求深入理解构建系统内部机制。
  - **Monitoring**：在无法直接查看源码的情况下，利用 `top`, `iostat`, `netstat`, `ps` 等命令行工具诊断运行时性能异常（如内存泄漏、I/O瓶颈）。
  - **Issue Resolving**：基于真实 GitHub Issue 修复 Java 和 Go 项目中的 Bug，区别于 SWE-bench 侧重 Python 的设定。
  - **Test Generation**：仅根据 Bug 描述编写回归测试，生成的代码需满足 "Fail-to-Pass" 标准（在 Bug 版本失败，修复版本通过）。
- **End-to-End Pipeline**：设计了 18 个全链路流水线任务，强制 Agent 串行完成 Build -> Monitor -> Fix -> Test，任一环节失败则整个任务终止。
- **严格防作弊机制**：移除所有 Git Metadata（如 `git log`），防止 Agent 通过历史提交记录直接获取修复方案；基于 **TerminalBench** 标准格式构建，确保环境隔离与可复现性。

#### 📊 实验数据/关键结论

- **全链路全军覆没**：在 End-to-End Pipeline 任务中，所有受测 SOTA 模型（包括 Claude-4-Sonnet, OpenAI o4-mini, DeepSeek-V3.1）的成功率均为 **0%**。
- **单阶段短板明显**：
  - **构建任务**：表现最好的 Claude Code 成功率仅为 **51.85%**。
  - **监控任务**：Claude Code 成功率仅为 **20.56%**，表现出对动态系统状态理解的极度匮乏。
  - **问题修复**：Claude-4-Sonnet + OpenHands 在 Java/Go 任务上修复率仅为 **23.87%**，相比 Python 任务出现断崖式下跌。
- **注意力涣散**：Agent 在监控长周期日志流时，往往过度关注早期观测结果，忽略了后续渐进式出现的关键异常信号（如缓慢的内存泄漏）。

#### 💡 独家洞察/局限性

- **"环境地狱"挑战**：实验证明，解决构建环境和配置问题是 AI 目前最薄弱的环节，这比单纯的代码生成更复杂，需要 Agent 具备处理模糊错误和复杂依赖解析的能力。
- **语言偏差效应**：AI Agent 在 Java/Go 等编译型语言上的表现远不如 Python，这暗示了当前大模型预训练数据中 Python 的主导地位可能限制了其在传统后端工程领域的泛化能力。
- **长程推理瓶颈**：0% 的全链路成功率表明，虽然 Agent 在单点任务上表现尚可，但在需要保持上下文、跨阶段推理的长任务链中，尚无法胜任真实生产环境的运维工作。

### 6. [AR3D-R1团队] 文本生成3D: Hi-GRPO分层架构与MME-3DR评测基准
**来源**: 量子位 | **时间**: 2026-02-27 09:24

> 🎯 **一句话摘要**：本文首次系统性将强化学习引入文本到3D生成任务，提出Hi-GRPO分层训练范式与MME-3DR评测基准，解决了3D视角不一致与奖励稀疏难题，显著提升了复杂文本描述下的生成质量。

#### 🔹 核心技术/实现逻辑

*   **奖励模型设计**：研究发现**人类偏好评分（HPS v2.1）** 是决定质量下限的核心，应以此为主，辅以语义对齐与美学质量。创新性地使用通用多模态大模型（Qwen2.5-VL）替代专用模型评估**3D几何一致性**，因其对空间关系理解更广泛，效果更鲁棒。
*   **算法选择与优化**：系统对比 GRPO、DAPO、GSPO 后，确认 **Token级Loss平均化**（源自DAPO）最适配3D的长程依赖特征；序列级操作在3D任务中因信号稀疏而收益极低。必须保留 **KL散度** 正则化防止策略漂移，并引入 **动态采样** 技巧以稳定训练曲线。

    *   **Stage 1（粗粒度）**：基于 Chain-of-Thought 生成高层语义与粗糙几何，奖励聚焦几何一致性。
    *   **Stage 2（细粒度）**：基于Stage 1生成精细纹理，奖励聚焦外观质量与部件完整性。
*   **数据策略**：数据多样性比训练时长更重要。数据翻倍有效，但过度迭代（如轮数翻三倍）会导致模型在偏好特征上过拟合，泛化能力下降。

*   **CLIP Score**: 22.7 -> 29.3 (+29%)
*   **Kernel Distance**: 降低约 37% (几何分布更逼真)
*   **Benchmark**: 在自建的 **MME-3DR**（249个复杂对象）和 Toys4K 上均超越 Trellis 等 SOTA 方法。

#### 💡 独家洞察/局限性

*   **技术点评**：该工作最大的价值在于打破2D RL经验主义，建立了针对3D生成的系统性研究框架。提出的**分层RL范式**（Hi-GRPO）将结构先验直接编码进训练过程，是解决3D长程依赖的关键。
*   **工程启示**：在没有现成3D几何专用Reward Model的情况下，利用通用VLM（如Qwen2.5-VL）进行多视角一致性评估是一个实用的工程Trick。

### 9. [Google] 元控制器：利用残差流操控实现内部强化学习与分层决策
**来源**: 新智元 | **时间**: 2026-02-27 05:55
**价值**: 🌟🌟🌟 **标签**: [强化学习] [模型架构] [Google Research] [论文解读]

> 🎯 **一句话摘要**：谷歌提出一种元控制器架构，通过操控预训练模型的残差流发现时序抽象动作，在稀疏奖励环境下实现高效探索，解决了传统RL难以进行长序列规划的问题。

#### 🔹 核心技术/实现逻辑

- **元控制器架构**：
  该研究引入了一个“元控制器”，它不直接优化输出token，而是接收基模型的**残差流**并生成一系列简单的“内部控制器”。每个内部控制器对应一个**时序抽象动作**，即跨越多个时间步的子程序（如“移动到蓝色方块”），并自带终止条件。

- **内部强化学习**：
  不同于在原始动作空间进行微调，该方法在元控制器发现的抽象动作空间中进行学习。通过变分推理，元控制器自动识别有意义的行为模块，大幅减小了搜索空间，使得在稀疏奖励环境下的学习成为可能。

- **“觉醒-睡眠”训练循环**：
  研究实现了 Schmidhuber 的理论框架，分为两个阶段：
  1. **睡眠阶段**：基模型通过自监督学习（下一个token预测）构建世界模型，学习潜在目标和时间抽象表征。
  2. **觉醒阶段**：训练元控制器操控基模型的内部残差流，生成有意义的抽象动作策略。

- **关键工程细节**：
  实验表明，在“觉醒”阶段训练元控制器时，必须**冻结预训练的基础模型**。如果两者同时训练（共训练），模型会收敛到退化解，无法发现有意义的时间抽象。这证明了分阶段训练的必要性。

#### 📊 实验数据/关键结论

- **任务表现**：在需要按顺序访问特定颜色的“组合式任务”中，该方法能够成功执行，而传统 token-by-token 探索难以完成。
- **对比结果**：在需要组合泛化的任务中，内部强化学习的成功率显著高于所有基线方法，包括先前最先进的分层强化学习方法 **CompILE**。
- **对齐验证**：元控制器学习到的开关模式与真实的子目标切换完美对齐，尽管模型从未接收过子目标标签，证明了其内部自发形成了类似“选项”的分层结构。

#### 💡 独家洞察/局限性

- **技术点评**：这项研究标志着AI研究从单纯优化输出转向操控模型内部认知过程。与稀疏自编码器（SAEs）等解释性方法相比，元控制器不仅能解释，还能通过干预残差流直接控制行为，具备内部记忆能力。
- **应用场景**：特别适合机器人控制（无需手工编码拆解任务）、数学推理（自动拆解复杂问题）等需要长期规划和组合推理的场景。
- **局限性**：文章主要展示了迷宫环境下的实验，在更开放或高维的真实世界场景中的泛化能力仍有待验证。

### 11. [DeepMind] AlphaEvolve: LLM作为遗传算子的自动算法发现
**来源**: 新智元 | **时间**: 2026-02-27 17:00
**价值**: 🌟🌟🌟 **标签**: [前沿研究] [算法设计] [LLM应用]
**链接**: https://mp.weixin.qq.com/s/KB66x5hlNa1W-Mbp5GjN3g

> 🎯 **一句话摘要**：DeepMind 提出AlphaEvolve框架，利用LLM（Gemini）作为遗传算子对算法源代码进行变异和自然选择，成功发现了超越人类直觉的博弈论算法（CFR与PSRO变体），在11个基准测试中全面碾压人类设计的经典方案。

#### 🔹 核心技术/实现逻辑

- **代码即基因组**：AlphaEvolve 不再局限于超参数调优，而是将算法的完整源代码视为基因组。搜索空间从有限的参数扩展到了LLM可理解的任意可执行代码空间。

- **LLM作为遗传算子**：使用 Gemini 1.5 充当“基因工程师”，执行语义级别的变异，而非传统的比特翻转。变异操作包括：重写逻辑控制流、修改符号操作、改变遗憾更新规则或权重调度策略。

- **进化循环机制**：
  1. **初始化**：从人类设计的经典算法（如 CFR+, DCFR）种子开始。
  2. **变异**：利用LLM生成新的算法变体。
  3. **评估**：在博弈论基准游戏上计算每个变体的**可利用度**（Exploitability，即偏离纳什均衡的程度，越低越好）作为适应度。
  4. **选择**：保留适应度最高的算法进入下一代。

- **目标算法家族**：主要针对不完全信息博弈中的两大核心算法进行进化：
  - **CFR (Counterfactual Regret Minimization)**：反事实遗憾最小化。
  - **PSRO (Policy Space Response Oracles)**：策略空间响应预言。

#### 📊 实验数据/关键结论

- **全面超越人类基线**：在 11 个博弈论测试场景中，AI 进化出的算法在收敛速度和最终性能上均优于人类几十年设计的最优方案（包括 CFR+, DCFR, PCFR+ 等）。在几乎所有测试博弈中，新算法的可利用度均显著低于人类算法。

- **VAD-CFR (Variable Adjusted Dynamic CFR)**：进化出的一种 CFR 变体。
  - **特性**：采用了反直觉的机制，如在局势混乱时果断遗忘旧经验、在发现好招时加倍下注、前 500 轮纯学习不进行策略平均。
  - **优势**：在早期迭代中极快地降低了可利用度。

- **SHOR-PSRO**：进化出的一种 PSRO 变体。
  - **特性**：实现了“训练与考试分离”，训练时追求稳健，考试时追求精准；采用了前期大胆试探、后期精准收网的策略。

#### 💡 独家洞察/局限性

- **范式转换**：该研究证明了算法设计本质上是一个搜索问题，而非纯粹依赖人类数学直觉的创造过程。AI 能够跳出“人类可解释”或“数学优雅”的局限，探索出极其有效但人类难以想象的“黑盒”数学机制。

- **局限性**：虽然发现了新算法，但部分进化出的机制（如 VAD-CFR 的特定遗忘规则）目前缺乏理论上的数学解释（即为什么有效），属于“经验主义”的胜利。

- **未来方向**：DeepMind 计划将此框架应用于深度强化学习（RL）智能体的完整架构设计，以及探索合作博弈中的机制发现。

*   **规格覆盖**：支持分辨率跨度从 **512px**（快速模式）到 **4K**（高精模式）。
